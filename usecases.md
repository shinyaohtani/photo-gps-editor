# ユースケース一覧

このドキュメントは PhotoGPSEditor の UX 設計のためのユースケースです。
各ユースケースは「誰が」「何を」「なぜ」「どうなるべきか」に加え、
考えうるユーザ体験の選択肢と、その中での最高の体験を記述します。

---

## 起動・初回体験

### UC-01: 初めてアプリを開いた
- **誰が**: このアプリを初めて使うユーザ
- **状況**: アプリが何をするものか知らない。画面に何もない
- **伝えるべきこと**: このアプリは「GPS のない写真に、別カメラの位置情報を使って GPS を付けるツール」であること。使うには 2 種類のデータが必要であること
- **あるべき姿**: 開いた瞬間に目的と最初の手順がわかる。迷わず次のアクションを起こせる
- **ユーザ体験の可能性リスト**:
  1. 空のウィンドウにツールバーだけ。ユーザは何をすればいいかわからない
  2. 中央に「ファイルを開いてください」とだけ書かれている。目的は伝わらない
  3. ウェルカム画面にアプリの説明と 2 つのボタン。読めばわかるが、読まない人もいる
  4. ドラッグ＆ドロップゾーンが 2 つ並んでおり、「ここに JSON を落とす」「ここにフォルダを落とす」と書かれている。視覚的に何をすればいいかが一目でわかる
  5. 起動直後にファイル選択ダイアログが自動で開く。意図がわからず戸惑う
- **最高のユーザ体験**: 画面中央に大きなドロップゾーンが 2 つ並んでいる。左は「iPhone の位置記録 JSON をここにドロップ」、右は「Canon 写真フォルダをここにドロップ」。それぞれにアイコンと 1 行の説明がある。ドロップの代わりにクリックしても NSOpenPanel が開く。画面上部には「GPS のない写真に、別カメラの位置情報から GPS を付けるツールです」と 1 行だけ書かれている。ユーザは読まなくても、ドロップゾーンの形状とラベルだけで次のアクションがわかる

### UC-02: 2 つ目のウィンドウを開いた
- **誰が**: Cmd+N で新しいウィンドウを開いたユーザ
- **状況**: すでに 1 つ目のウィンドウで作業中、または起動直後
- **問題**: 同じ状態のウィンドウが 2 つ開き、どちらが本物かわからない
- **あるべき姿**: このアプリはシングルウィンドウで十分。新規ウィンドウを禁止するか、開いても独立した空の状態で起動し混乱しないようにする
- **ユーザ体験の可能性リスト**:
  1. 何もしない。同じ状態のウィンドウが増殖する
  2. Cmd+N を無効化する。新しいウィンドウは開かない
  3. 新しいウィンドウは独立した空セッションで開く。複数プロジェクトを並行作業できる
  4. Cmd+N を押すと既存ウィンドウが前面に来る。新規ウィンドウは作らない
- **最高のユーザ体験**: Cmd+N は既存ウィンドウを前面に持ってくるだけにする。このアプリは 1 つの写真セットを 1 つのウィンドウで扱う単機能ツールであり、複数ウィンドウは混乱しか生まない。ウィンドウを閉じて Dock アイコンをクリックした場合も同様にウィンドウを再表示する

### UC-03: 前回の作業を再開したい
- **誰が**: 2 回目以降の起動ユーザ
- **状況**: 前回読み込んだ JSON とフォルダがディスク上に残っている
- **あるべき姿**: 前回のパスを記憶しており、起動時に自動で読み込む。読み込み中であることが見てわかる
- **ユーザ体験の可能性リスト**:
  1. 毎回空で起動。ユーザが毎回手動で開き直す
  2. UserDefaults にパスを保存し、起動時に自動読み込み。ただし読み込み中は何も見えない
  3. 起動時に前回のパスを表示し「これを開きますか？」と確認ダイアログを出す
  4. 起動時に自動読み込みを開始し、ウェルカム画面上に進捗を表示する。読み込み完了と同時に地図画面に切り替わる
- **最高のユーザ体験**: 起動するとウェルカム画面のドロップゾーンに前回のパスが薄く表示されており、同時にバックグラウンドで読み込みが始まっている。進捗が見える。読み込みが終わると自然に地図画面に遷移する。ユーザは何も操作せずに前回の続きから始められる。別のデータを使いたければ、遷移前にドロップゾーンに別のファイルを落とせばそちらが優先される

### UC-04: 前回のファイルが消えていた
- **誰が**: 2 回目以降の起動ユーザ
- **状況**: 前回使ったファイルやフォルダが移動・削除された
- **あるべき姿**: 「前回のファイルが見つかりません」と明示し、再選択を促す。黙って空で起動しない
- **ユーザ体験の可能性リスト**:
  1. 黙って空で起動。ユーザは前回のデータがどこへ行ったかわからない
  2. エラーダイアログを出す。閉じないと先に進めない
  3. ウェルカム画面のドロップゾーン上に「前回のファイルが見つかりません」と表示し、新しいファイルのドロップまたは選択を促す
  4. ドロップゾーンに前回のパスを取り消し線付きで表示し、横に「見つかりません — 別のファイルを選んでください」と添える
- **最高のユーザ体験**: ドロップゾーンに前回のパスが取り消し線つきで残っており、「見つかりません」と赤字で添えられている。何が起きたか一目でわかり、ドロップゾーンにそのまま新しいファイルを落とせばよい。モーダルダイアログは使わない。作業の流れを止めない

## データ読み込み

### UC-05: 参照 JSON を選びたい
- **誰が**: ワークフローを始めるユーザ
- **状況**: iPhone 側の位置情報付き写真の JSON を持っている
- **伝えるべきこと**: 「参照 JSON」とは何か、どこから取得するものか
- **あるべき姿**: ファイル選択後、読み込んだ件数と地図上の分布が即座に見える
- **ユーザ体験の可能性リスト**:
  1. ツールバーのボタンを押して NSOpenPanel から選ぶ。説明なし
  2. ドロップゾーンに JSON をドラッグ＆ドロップ。ゾーンのラベルが説明を兼ねる
  3. ドロップゾーンをクリックすると NSOpenPanel が開く。ドロップもできる
  4. ドロップゾーンに「iPhone の写真アプリからエクスポートした JSON」と具体的な入手手順が書いてある
- **最高のユーザ体験**: ドロップゾーンに JSON をドラッグすると、ゾーンがハイライトし「ここにドロップ」と表示が変わる。ドロップすると即座に読み込みが始まり、ゾーン内に「2,291 枚の参照写真を読み込み中…」と進捗が出る。完了するとゾーンにチェックマークとファイルパスが表示される。クリックでも NSOpenPanel を開ける

### UC-06: Canon フォルダを選びたい
- **誰が**: ワークフローを始めるユーザ
- **状況**: GPS を付けたい Canon 写真のフォルダを持っている
- **伝えるべきこと**: どのフォルダを選べばよいか（DCIM の中のフォルダ、など）
- **あるべき姿**: フォルダ選択後、読み込んだ件数と GPS 有無の内訳が見える
- **ユーザ体験の可能性リスト**:
  1. ツールバーのボタンから NSOpenPanel で選ぶ
  2. ドロップゾーンにフォルダをドラッグ＆ドロップ
  3. Finder からフォルダを直接ウィンドウにドロップ
- **最高のユーザ体験**: Finder からフォルダをドロップゾーンにドラッグすると、ゾーンがハイライトする。ドロップすると「500 枚の写真を読み込み中…」と進捗が出る。完了すると「497 枚 JPG + 3 本 MP4、うち GPS 付き 0 枚」のように内訳が表示される

### UC-07: 読み込み中に何が起きているか知りたい
- **誰が**: Load ボタンを押した直後のユーザ
- **状況**: 500 枚以上の写真を読み込んでおり数秒かかる
- **あるべき姿**: プログレスバーまたはスピナーと「Loading 234/500…」のような進捗が見える。UI がフリーズしていないことがわかる
- **ユーザ体験の可能性リスト**:
  1. スピナーだけ回る。何枚中何枚かわからない
  2. 「読み込み中…」の文字だけ。進捗不明
  3. スピナー + 件数カウント（「Loading 234/500…」）
  4. プログレスバー + 件数カウント + 経過時間
- **最高のユーザ体験**: ドロップゾーン内にプログレスバーが表示され、「234 / 500 枚を読み込み中…」とリアルタイムで更新される。EXIF 読み取りは 1 枚ずつではなくバッチで進むが、ユーザには枚数の進捗として見える。フリーズしていないことが視覚的に確実にわかる

### UC-08: 読み込むデータを差し替えたい
- **誰が**: 作業途中で別のフォルダに切り替えたいユーザ
- **状況**: 間違ったフォルダを選んだ、または別の旅行のデータに切り替えたい
- **あるべき姿**: 再度 Load ボタンから別のファイル/フォルダを選べる。前のデータは破棄され、新しいデータに置き換わる
- **ユーザ体験の可能性リスト**:
  1. ツールバーの Load ボタンを再度押す。確認なしで即座に差し替わる
  2. 差し替え前に「現在のデータは破棄されます。よろしいですか？」と確認
  3. 地図画面のまま差し替え可能。ツールバーのボタンがいつでも有効
- **最高のユーザ体験**: ツールバーの Load ボタンはいつでも押せる。未保存の GPS 変更がある場合だけ「未保存の変更があります。破棄しますか？」と確認が出る。変更がなければ確認なしで即座に差し替わる。差し替え後は補間前の状態に戻る

## GPS 補間

### UC-09: GPS を自動補間したい
- **誰が**: 両方のデータを読み込み終えたユーザ
- **状況**: Canon 写真に GPS がなく、iPhone の記録を基準に付けたい
- **あるべき姿**: Interpolate を押すと、GPS のない Canon 写真に一括で GPS が付く。何枚に付いたかがわかる。地図上にピンが出現する
- **ユーザ体験の可能性リスト**:
  1. ボタンを押して待つ。完了メッセージがステータスバーに出る
  2. 両方読み込み完了した瞬間に自動で補間が走る。ボタンを押す必要がない
  3. 補間後、地図が自動で全ピンが見える範囲にズームし、結果の概要がポップアップで出る
  4. 補間ボタンを押すと、まずタイムゾーンの確認ダイアログが出て、確認後に補間が走る
- **最高のユーザ体験**: 両方のデータが揃うと、ツールバーの Interpolate ボタンが脈動するようにハイライトし「次はこれを押してください」と視覚的に誘導する。押すと補間が走り、地図が全ピンを含む範囲にズームし、「450 枚に GPS を付与しました」とステータスに表示される。GPS が付いた瞬間にオレンジのピンが地図上に出現するのが見える

### UC-10: タイムゾーンが合っているか不安
- **誰が**: 海外旅行の写真を扱うユーザ
- **状況**: Canon のカメラ時計が日本時間のまま、旅行先はイタリア
- **伝えるべきこと**: タイムゾーン設定が何に影響するか。間違えるとすべての位置がずれること
- **あるべき姿**: 補間前にタイムゾーンの確認を促す。補間後に結果が明らかにおかしければ「タイムゾーンを確認してください」とガイドする
- **ユーザ体験の可能性リスト**:
  1. TZ ピッカーがツールバーにあるだけ。気づかない人は気づかない
  2. 補間ボタンを押した時に「タイムゾーンは JST +9 です。合っていますか？」と確認
  3. Canon と iPhone の撮影時刻を比較し、明らかにずれている場合に自動で警告
  4. 補間後に Canon 写真のピンと iPhone 参照点のピンが時系列で大きくずれている場合、「タイムゾーンが間違っている可能性があります」と警告
- **最高のユーザ体験**: Canon 写真読み込み時に、Canon の最初の撮影時刻と最後の撮影時刻を、現在の TZ 設定で UTC に変換し、iPhone 参照写真の時間範囲と比較する。重なりが少ない場合、ドロップゾーンまたはステータスに「Canon の時間範囲と参照写真の時間範囲が大きくずれています。タイムゾーン設定を確認してください」と黄色い警告を出す。ユーザが TZ を変更すると即座に再評価し、警告が消える

### UC-11: 一部の写真だけ再補間したい
- **誰が**: 補間後に一部の位置がずれていることに気づいたユーザ
- **状況**: 手で動かすより、選択した写真だけもう一度補間し直したい
- **あるべき姿**: 写真を選択した状態で Interpolate を押すと、選択分だけ再補間される。全体には影響しない
- **ユーザ体験の可能性リスト**:
  1. Interpolate ボタンは常に全件対象。部分再補間はできない
  2. 選択があると Interpolate ボタンのラベルが「Re-Interpolate Selected」に変わる
  3. Interpolate ボタンの tooltip が選択状態に応じて変わり、対象枚数が見える
- **最高のユーザ体験**: 写真を選択すると、ツールバーの Interpolate ボタンの tooltip が「選択中の N 枚だけを再補間します」に変わる。押すと選択分だけ再計算され、他のピンは動かない。Undo で元に戻せる。選択を解除すると tooltip は通常の「GPS なしの写真を補間します」に戻る

### UC-12: Canon 写真にすでに GPS が入っている
- **誰が**: 以前に GPS を書き込み済みのフォルダを開いたユーザ
- **状況**: 再読み込みしたが、すでに全部 GPS がある
- **あるべき姿**: 「GPS 付き N 枚読み込みました」と表示。Interpolate は GPS なしの写真だけを対象にする。全部 GPS 済みなら「補間対象なし」と伝える
- **ユーザ体験の可能性リスト**:
  1. 何も言わず、Interpolate しても 0 件。ユーザは混乱する
  2. 読み込み完了時に「全 500 枚に GPS が付いています」と表示
  3. Interpolate ボタンの tooltip に「補間対象なし」と出る。ボタンは有効だが押しても 0 件
  4. 読み込み時に GPS 付きの写真は既存座標で地図にピンを表示する
- **最高のユーザ体験**: フォルダ読み込み完了時に「497 枚 JPG + 3 本 MP4、全 500 枚に GPS 付き」と表示され、既存の GPS 座標で地図上にピンが即座に表示される。Interpolate の tooltip には「GPS なしの写真がありません。選択した写真を再補間したい場合は、先に写真を選択してください」と出る。ユーザは既存 GPS を確認しながら、必要な部分だけ選択して再補間できる

## 地図操作・選択

### UC-13: 地図上でピンの写真を確認したい
- **誰が**: 補間結果を確認しているユーザ
- **状況**: ピンが何の写真かわからない
- **あるべき姿**: ピンにマウスを乗せるとプレビュー画像が大きく表示される。矢印キーで前後の写真へ移動できる
- **ユーザ体験の可能性リスト**:
  1. クリックして選択し、サイドバーでサムネイルを見る
  2. ホバーで小さなツールチップにサムネイルが出る
  3. ホバーで画面中央に大きなプレビューパネルが出る。マウスを離すと消える
  4. ホバーで大きなプレビュー + ファイル名 + 撮影時刻がオーバーレイ表示される
- **最高のユーザ体験**: ピンにマウスを乗せると 0.1 秒後に画面中央に大きなプレビューが表示される。写真の中身がはっきり見え、建物や風景で場所を確認できる。矢印キーで時系列順に前後の写真に切り替えられる。Esc で閉じる。Option キーを押している間はプレビューが出ないので、移動操作中に邪魔にならない

### UC-14: 複数のピンをまとめて選択したい
- **誰が**: ある区間のピンを一括で動かしたいユーザ
- **状況**: レストランで撮った 30 枚が少しずれているので、まとめて直したい
- **あるべき姿**: ドラッグで矩形選択。Shift+ドラッグで追加選択。選択数がすぐわかる
- **ユーザ体験の可能性リスト**:
  1. 1 枚ずつ Shift+クリックで選択。30 枚は辛い
  2. ドラッグで矩形選択。選択数はサイドバーで確認
  3. ドラッグ中にリアルタイムで矩形内のピン数が表示される
  4. ドラッグで矩形選択 + 矩形の端にリアルタイムで「23 枚」と出る
- **最高のユーザ体験**: 空白部分をドラッグすると青い矩形が描かれ、矩形内のピンがリアルタイムでハイライトされる。マウスを離すと選択が確定し、サイドバーの選択数が即座に更新される。Shift を押しながらドラッグすると、既存の選択に追加できる。選択解除は空白をクリック

### UC-15: 選択したピンを移動したい
- **誰が**: 補間位置がずれている写真を正しい場所に動かしたいユーザ
- **状況**: ピンを選択済み
- **あるべき姿**: Option+ドラッグで選択中のピンが全部一緒に動く。通常ドラッグは矩形選択なので、誤操作で位置を動かすことがない
- **ユーザ体験の可能性リスト**:
  1. 選択したピンをそのままドラッグで移動。ただし矩形選択と操作が衝突する
  2. Option+ドラッグで移動。通常ドラッグと分離されて安全
  3. 移動中、ピンが半透明のゴーストとして移動先に表示される
  4. 移動中にピンに対応する写真のプレビューが出て、移動先が正しいか確認できる
- **最高のユーザ体験**: Option+ドラッグを開始すると、選択中のピンが全部一緒にマウスに追従する。ドラッグ中はプレビューが出ない（Option 押下中はプレビュー抑制）。マウスを離すと移動が確定する。Cmd+Z で即座に元に戻せる。移動対象は Canon 写真だけで、参照点は動かない

### UC-16: 同じ場所に重なった複数のピンを区別したい
- **誰が**: 密集地帯で特定の写真だけ選びたいユーザ
- **状況**: 同じ座標に 10 枚重なっている
- **あるべき姿**: 右クリックで写真一覧が出る。サムネイルとファイル名で識別でき、チェックボックスで個別に選択/解除できる
- **ユーザ体験の可能性リスト**:
  1. 右クリックでファイル名だけのリストが出る
  2. 右クリックでサムネイル付きリストが出る。チェックボックスで選択
  3. 右クリックでサムネイル付きリスト + 撮影時刻が出る
  4. ズームインすると重なったピンが自動で散開する（クラスター解除）
- **最高のユーザ体験**: 右クリックするとポップオーバーが開き、その座標にある全写真がサムネイル・ファイル名・撮影時刻付きで一覧される。チェックボックスで個別に選択/解除できる。ポップオーバーは閉じるまで開いたままなので、複数のチェックを入れてから閉じて、まとめて移動できる

### UC-17: 操作を間違えたので戻したい
- **誰が**: ピンを動かしすぎた、または間違った写真を動かしたユーザ
- **状況**: 直前の操作を取り消したい
- **あるべき姿**: Cmd+Z で無限に Undo。Shift+Cmd+Z で Redo。補間もドラッグ移動も対象
- **ユーザ体験の可能性リスト**:
  1. Undo なし。動かしたらやり直し
  2. 1 回だけ Undo。2 回前には戻れない
  3. 無限 Undo。ただし何を Undo したかわからない
  4. 無限 Undo + ステータスバーに「Undo: 5 枚の移動を元に戻しました」と表示
- **最高のユーザ体験**: Cmd+Z を押すと即座に直前の操作が取り消され、ステータスバーに「Undo: 5 枚の移動を元に戻しました」や「Undo: 補間を元に戻しました」と何を戻したかが表示される。Shift+Cmd+Z で Redo。回数制限なし。地図上のピンが元の位置に戻るアニメーションが見える

## サイドバー・リスト

### UC-18: 選択中の写真をリストで確認したい
- **誰が**: 地図で矩形選択したユーザ
- **状況**: 何枚選んだか、どの写真が含まれているか知りたい
- **あるべき姿**: 地図の選択とリストのハイライトが即座に同期する。選択数が見える。リストが選択写真の位置にスクロールする
- **ユーザ体験の可能性リスト**:
  1. 選択数だけ表示。リストは動かない
  2. 選択数 + リストのハイライトが変わるが、スクロールしない
  3. 選択数 + ハイライト + 最初の選択写真へ自動スクロール
  4. 選択数 + ハイライト + フィルタが自動で「Selected」に切り替わり、選択中の写真だけが見える
- **最高のユーザ体験**: 矩形選択を完了した瞬間に、サイドバーの選択数が更新され、リストのハイライトが同期する。リストは最初の選択写真の位置に自動スクロールする。フィルタを「Selected」に切り替えると選択中の写真だけが並び、全体像を確認できる。地図上でピンの選択を変えるたびにリストがリアルタイムで追従する

### UC-19: リストから写真を選んで地図で確認したい
- **誰が**: 特定の写真がどこに配置されたか確認したいユーザ
- **状況**: ファイル名はわかるが、地図上のどこかわからない
- **あるべき姿**: リストの写真をクリックすると、地図がその位置にズームする
- **ユーザ体験の可能性リスト**:
  1. リストクリックで選択されるが、地図は動かない
  2. リストクリックで地図がその写真の位置にパンする
  3. リストクリックで地図がズームインし、ピンがハイライトされる
  4. リストクリックで地図がズームイン + プレビューが自動表示される
- **最高のユーザ体験**: リストの写真をクリックすると、地図がその写真の位置にアニメーションでズームインし、対象のピンが選択状態（緑）になる。ズームレベルは周囲の参照点が 1〜2 個見える程度。ユーザは位置の妥当性をすぐ判断できる

### UC-20: GPS が付いていない写真だけ見たい
- **誰が**: 補間漏れがないか確認したいユーザ
- **状況**: 500 枚中、まだ GPS がない写真があるか知りたい
- **あるべき姿**: フィルタで「No GPS」に切り替えると、未付与の写真だけがリストに出る。0 件なら「すべて GPS 付与済み」とわかる
- **ユーザ体験の可能性リスト**:
  1. リストを全件スクロールして「No GPS」の赤字を探す
  2. フィルタ切り替えで「No GPS」の写真だけ表示
  3. フィルタ + 件数表示（「No GPS: 3 枚」）
  4. フィルタボタン自体に件数バッジが付いている
- **最高のユーザ体験**: フィルタの「No GPS」タブに件数が表示されている（「No GPS (3)」）。切り替えると未付与の写真だけが並ぶ。0 件のときはタブに「(0)」と出ており、切り替えると「すべての写真に GPS が付与されています」と中央に表示される。補間漏れがないことが即座に確認できる

## 保存

### UC-21: GPS を Canon 写真に書き込みたい
- **誰が**: 位置調整が終わったユーザ
- **状況**: 補間と微調整が完了し、EXIF に保存したい
- **あるべき姿**: Save EXIF を押すと、変更のある写真だけに GPS が書き込まれる。何枚保存したかがわかる。既に保存済みで変更なしなら「変更なし」と伝える
- **ユーザ体験の可能性リスト**:
  1. ボタンを押すと全件に書き込む。変更なしでも上書き
  2. 変更ありの写真だけに書き込む。件数をステータスに表示
  3. 変更ありの写真だけに書き込む + 確認ダイアログ
  4. 変更ありの写真だけに書き込む + プログレスバー + 完了後に件数表示
- **最高のユーザ体験**: Save EXIF を押すと「N 枚の Canon 写真に GPS を書き込みます。元のファイルは上書きされます。」という確認ダイアログが出る。OK を押すとプログレスバー付きで書き込みが進み、完了すると「245 枚に GPS を保存しました」と表示される。変更なしの場合は「未保存の変更はありません」と即座にフィードバック

### UC-22: 保存前に対象を確認したい
- **誰が**: 慎重なユーザ
- **状況**: 本当に全部書き込んでよいか不安
- **伝えるべきこと**: 何枚が書き込み対象か。Save は元ファイルを上書きすること
- **あるべき姿**: Save ボタンの tooltip に対象枚数が出る。押した後に確認ダイアログがあるとなお良い
- **ユーザ体験の可能性リスト**:
  1. tooltip に枚数だけ。確認なしで即保存
  2. tooltip に枚数 + 確認ダイアログ
  3. tooltip に枚数 + 確認ダイアログ + ダイアログ内に対象ファイル一覧
  4. Save ボタンにバッジで未保存枚数を表示 + 確認ダイアログ
- **最高のユーザ体験**: Save ボタンの tooltip には常に「変更あり N 枚」と表示されている。ボタンを押すと確認ダイアログが開き、「以下の N 枚の Canon 写真に GPS を書き込みます。元のファイルが上書きされます。」と表示される。ユーザは枚数を確認して OK を押す。キャンセルすれば何も起きない

## エラー・例外

### UC-23: exiftool がインストールされていない
- **誰が**: 初めて Save しようとしたユーザ
- **状況**: Homebrew で exiftool を入れていない
- **あるべき姿**: 「exiftool が見つかりません。brew install exiftool でインストールしてください」と明示する。黙って 0 件保存にしない
- **ユーザ体験の可能性リスト**:
  1. 黙って 0 件保存。ユーザは成功したと思う
  2. エラーダイアログに「exiftool が見つかりません」と表示
  3. エラーダイアログにインストール手順まで記載
  4. 起動時に exiftool の存在を確認し、なければウェルカム画面で案内する
- **最高のユーザ体験**: アプリ起動時に `/opt/homebrew/bin/exiftool` の存在をチェックする。見つからなければウェルカム画面の下部に黄色い警告バナーで「exiftool が見つかりません。GPS の保存には exiftool が必要です。ターミナルで brew install exiftool を実行してください。」と表示する。Save ボタンを押した時にも改めて確認し、なければ同じメッセージのダイアログを出す

### UC-24: 参照写真と Canon 写真の時間帯が重ならない
- **誰が**: 参照 JSON の時間範囲と Canon 写真の時間範囲がずれているユーザ
- **状況**: 参照は 3/11〜3/15 だが Canon は 3/20〜3/25 のデータ
- **あるべき姿**: 補間結果が全部同じ座標（最後の参照点）になったら、「時間帯が重なっていません」と警告する。タイムゾーン設定の確認も促す
- **ユーザ体験の可能性リスト**:
  1. 黙って補間する。全部同じ座標になっても何も言わない
  2. 補間後に「すべての写真が同じ位置に配置されました」と警告
  3. 補間前に時間範囲を比較し、重なりがなければ事前に警告
  4. 補間前に時間範囲を比較し、重なりがなければ補間を中止して警告。タイムゾーン変更を提案
- **最高のユーザ体験**: 両方のデータが揃った時点で、参照写真と Canon 写真の時間範囲を自動で比較する。重なりがない場合、ステータスに「参照写真の時間範囲 (3/11〜3/15) と Canon 写真の時間範囲 (3/20〜3/25) が重なっていません。タイムゾーン設定を確認するか、データが正しいか確認してください」と黄色い警告を表示する。Interpolate を押すと「時間範囲が重なっていませんが、続行しますか？」と確認が出る。ユーザは状況を理解した上で判断できる
